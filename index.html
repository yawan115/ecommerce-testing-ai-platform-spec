☁️Kiwi TCMS 部署（GCP）

推荐方案：Cloud Run（无服务器）+ Cloud SQL（PostgreSQL）

第一步：创建 Cloud SQL

bash
gcloud sql instances create kiwi-db \
  --database-version=POSTGRES_15 \
  --tier=db-f1-micro \
  --region=asia-east1 \
  --project=YOUR_PROJECT_ID

gcloud sql databases create kiwi --instance=kiwi-db
gcloud sql users create kiwi_user \
  --instance=kiwi-db --password=YOUR_SECURE_PASSWORD

第二步：推送镜像到 Artifact Registry

bash
docker pull kiwitcms/kiwi:latest
docker tag kiwitcms/kiwi:latest \
  asia-east1-docker.pkg.dev/PROJECT/kiwi/app:latest
docker push asia-east1-docker.pkg.dev/PROJECT/kiwi/app:latest

第三步：部署 Cloud Run

bash
gcloud run deploy kiwi-tcms \
  --image=asia-east1-docker.pkg.dev/PROJECT/kiwi/app:latest \
  --region=asia-east1 \
  --platform=managed \
  --add-cloudsql-instances=PROJECT:asia-east1:kiwi-db \
  --set-env-vars="KIWI_DB_HOST=/cloudsql/PROJECT:asia-east1:kiwi-db" \
  --set-secrets="KIWI_DB_PASSWORD=kiwi-db-password:latest" \
  --memory=1Gi \
  --allow-unauthenticated=false

第四步：初始化数据库

bash
gcloud run jobs create kiwi-migrate \
  --image=asia-east1-docker.pkg.dev/PROJECT/kiwi/app:latest \
  --command="python" --args="manage.py,migrate" \
  --region=asia-east1

gcloud run jobs execute kiwi-migrate

🔗Jira 集成

Jira → Kiwi（正向）

  1. Story 创建/更新触发 Webhook
  2. Cloud Function 调用 AI 生成测试用例
  3. 自动在 Kiwi 创建用例并关联需求

Kiwi → Jira（反向）

  1. 执行失败触发 Kiwi Webhook
  2. 自动在 Jira 创建 Bug Issue
  3. 关联原始 Story，附上日志截图

Jira Webhook 配置

配置项
URL:    https://REGION-PROJECT.cloudfunctions.net/jira-to-kiwi
Events: Issue Created, Issue Updated
Filter: project in (CHECKOUT, INVENTORY, CUSTOMER)
        AND issuetype = Story

Cloud Function 核心代码

python — jira_to_kiwi/main.py
import functions_framework
from anthropic import Anthropic
import requests

@functions_framework.http
def handle_jira_webhook(request):
    payload = request.get_json()
    story = payload['issue']
    title = story['fields']['summary']
    description = story['fields']['description']
    acceptance_criteria = story['fields'].get('customfield_10016', '')

    # AI 生成测试用例（使用 Haiku 省成本）
    client = Anthropic()
    response = client.messages.create(
        model="claude-haiku-3-5",
        max_tokens=2000,
        messages=[{
            "role": "user",
            "content": f"""
            根据以下用户故事生成测试用例（JSON格式）。
            包括：正向场景、负向场景、边界场景、API测试要点。

            Story: {title}
            描述: {description}
            验收标准: {acceptance_criteria}
            """
        }]
    )

    test_cases = parse_ai_response(response.content[0].text)
    for tc in test_cases:
        create_kiwi_testcase(tc, story['key'])

    return {'status': 'ok', 'created': len(test_cases)}

⚙️GitHub Actions 集成

各团队保留现有测试框架，只需在 CI 末尾加上统一上报步骤。

团队现有测试框架上报方式
Checkout Pytest pytest --junit-xml=results.xml
Inventory Newman (Postman) newman run --reporters junit
Customer Playwright playwright test --reporter=junit

统一 GitHub Actions 模板

yaml — .github/workflows/test-and-report.yml
name: Test & Report to Kiwi TCMS
on: [push, pull_request]

env:
  KIWI_URL: ${{ secrets.KIWI_URL }}
  KIWI_USERNAME: ${{ secrets.KIWI_USERNAME }}
  KIWI_PASSWORD: ${{ secrets.KIWI_PASSWORD }}
  KIWI_PLAN_ID: "checkout-sprint-42"   # 各团队修改此处

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      # ↓ 各团队保留自己的测试步骤
      - name: Run Tests
        run: |
          pip install pytest pytest-junit
          pytest --junit-xml=results.xml

      # ↓ 统一上报步骤（所有团队相同，无需修改）
      - name: Report Results to Kiwi TCMS
        if: always()
        run: |
          pip install tcms-api
          python -c "
          from tcms_api import TCMS
          tcms = TCMS()
          tcms.upload_junit('results.xml', plan_id='$KIWI_PLAN_ID')
          "

      - name: Upload Artifacts on Failure
        if: failure()
        uses: actions/upload-artifact@v4
        with:
          name: test-results
          path: results.xml

核心 SQL 查询

sql — 各团队需求覆盖率
SELECT
  p.name                                              AS team,
  COUNT(DISTINCT r.id)                                AS total_requirements,
  COUNT(DISTINCT tc.id)                               AS covered,
  ROUND(COUNT(DISTINCT tc.id) * 100.0
        / NULLIF(COUNT(DISTINCT r.id), 0), 1)         AS coverage_pct
FROM testplans_testplan p
JOIN requirements r  ON r.plan_id  = p.id
LEFT JOIN testcases_testcase tc ON tc.requirement_id = r.id
GROUP BY p.name
ORDER BY coverage_pct;
