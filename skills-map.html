可用 Skills 详细分析（10 个强相关）
Phase 2 · 用例生成
核心

bdd-testing

E2E-first BDD 测试工作流。强调先设计端到端场景，再到单元测试。内置边界条件检查清单和发布门禁（E2E 全过才能发）。

✓ 让 AI 为下单/库存/用户场景生成 Gherkin 格式 E2E 用例
Phase 3 · 质量门禁
核心

llm-as-qa-judge ⭐

我们刚发布的 skill。100 分制评测 AI 生成的测试用例，场景覆盖 + 步骤清晰度 + AC 对齐 + 无冗余四个维度。

✓ 测试用例提交 Kiwi 前必过的 AI 质量门禁
Phase 2 · 用例生成
核心

ikea-api-automation-test

自动生成 Python API 测试框架，包含目录结构、pytest 模板、测试数据文件、conftest 等完整脚手架。

✓ Checkout/Inventory API 接口测试代码一键生成
Phase 1 · 需求规划
核心

ikea-jira

JQL 查询、创建/更新 Issue、管理状态、查看看板。无需 MCP Hub，直接 PAT 鉴权，脚本即用。

✓ 从 Jira Sprint 拉取 Story AC → 喂给 AI 生成测试用例
Phase 1 · 需求规划
高度相关

dtc-userstory-generate

基于 CNA 产品知识库，从一句话生成完整 Jira Story + AC，并自动创建到 Jira。内置 DTC 功能域映射表。

✓ PM 描述功能 → AI 生成含 AC 的 Story → 直接创建 Jira
Phase 2 · 数据准备
高度相关

ikea_ciam_private_test_account_generator

生成 IKEA CIAM 测试账号数据，包含手机号、partyUid、会员卡号、id_token 等关键字段，CSV 格式输出。

✓ 下单/用户登录场景的测试账号批量准备
Phase 5 · 监控诊断
高度相关

ikea-grafana

LogCLI + Loki 日志查询，支持 LogQL 语法，按服务名/时间范围过滤。可查询测试运行后的错误日志。

✓ 自动化测试失败后，Loki 查日志定位根因
Phase 5 · 监控诊断
高度相关

cdhf-oncall-doctor

Incident 根因诊断助手，Jira 拉取 Issue → 搜历史同类 → Grafana 日志串联 → 输出结构化诊断报告。

✓ 测试发现 Incident 后，自动串联 Jira+日志输出根因报告
Phase 4 · CI/CD
高度相关

ikea-github

GitHub 仓库操作：PR 创建、分支管理、Actions 触发。可用于 CI 流水线中自动提交测试结果。

✓ 测试通过后自动创建 PR，触发 GitHub Actions 流水线
Phase 3 · 质量门禁
可选

ikea-security

AI 代码安全审计方法论，覆盖密钥泄露、注入攻击、XSS、OWASP Top 10。无需安装工具，代码审阅即用。

✓ 发布前对测试代码和业务代码做安全扫描
