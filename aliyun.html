✅为什么选阿里云部署

⚡
低延迟

业务代码与 Kiwi TCMS 同在阿里云，CI/CD 上报结果走内网，延迟极低

数据不出境

测试数据、覆盖率报告全留在阿里云，满足数据合规要求

💰
统一账单

无跨云流量费，阿里云资源包/折扣统一适用，账单更简单

🛠
运维简单

一个云平台统一运维，不需要跨云权限管理和网络打通

1

创建 RDS for PostgreSQL

在阿里云控制台 → RDS → 创建实例，选择 PostgreSQL 15，或使用 CLI：

bash — 阿里云 CLI
aliyun rds CreateDBInstance \
  --Engine PostgreSQL \
  --EngineVersion 15.0 \
  --DBInstanceClass pg.n2.small.1 \
  --DBInstanceStorage 20 \
  --DBInstanceStorageType cloud_essd \
  --RegionId cn-hangzhou \
  --ZoneId cn-hangzhou-h \
  --VpcId vpc-xxxx \
  --VSwitchId vsw-xxxx

# 创建数据库和用户
aliyun rds CreateDatabase \
  --DBInstanceId rm-xxxx \
  --DBName kiwi \
  --CharacterSetName UTF8

aliyun rds CreateAccount \
  --DBInstanceId rm-xxxx \
  --AccountName kiwi_user \
  --AccountPassword YourSecurePassword123 \
  --AccountType Normal

🌐Nginx 反向代理 + HTTPS

申请阿里云免费 SSL 证书（DV 单域名），或使用 Let's Encrypt。

申请免费 SSL 证书

🔐 阿里云控制台 → SSL 证书 → 免费证书 → 申请（每年 20 个免费额度），下载 Nginx 格式证书

Nginx 配置

nginx — /etc/nginx/conf.d/kiwi.conf
server {
    listen 80;
    server_name kiwi.your-domain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name kiwi.your-domain.com;

    ssl_certificate     /etc/nginx/ssl/kiwi.crt;
    ssl_certificate_key /etc/nginx/ssl/kiwi.key;
    ssl_protocols       TLSv1.2 TLSv1.3;

    client_max_body_size 100M;

    location / {
        proxy_pass         http://127.0.0.1:8080;
        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto https;
        proxy_read_timeout 60s;
    }

    location /static/ {
        alias /data/kiwi/static/;
        expires 30d;
    }
}

函数计算 FC 部署

bash — 安装 Serverless Devs 并部署
# 安装 Serverless Devs
npm install -g @serverless-devs/s

# 初始化函数
s init python3.10-http -d jira-to-kiwi
cd jira-to-kiwi

# 安装依赖
pip install anthropic requests -t ./

# 部署
s deploy --access aliyun
python — jira_to_kiwi/index.py
import json, os
import requests
from anthropic import Anthropic

def handler(environ, start_response):
    body = environ['wsgi.input'].read()
    payload = json.loads(body)
    story = payload.get('issue', {})

    title = story.get('fields', {}).get('summary', '')
    description = story.get('fields', {}).get('description', '') or ''
    ac = story.get('fields', {}).get('customfield_10016', '') or ''

    # AI 生成（可替换为通义千问节省成本）
    client = Anthropic(api_key=os.environ['ANTHROPIC_API_KEY'])
    resp = client.messages.create(
        model="claude-haiku-3-5",
        max_tokens=2000,
        messages=[{
            "role": "user",
            "content": f"""
            电商平台QA工程师角色，根据Story生成JSON格式测试用例。
            包括：正向/负向/边界/API测试场景。
            Story: {title}
            描述: {description}
            验收标准: {ac}
            """
        }]
    )

    test_cases = parse_response(resp.content[0].text)
    kiwi_url = os.environ['KIWI_URL']

    for tc in test_cases:
        requests.post(f"{kiwi_url}/api/v6/testcases/", json=tc,
                      auth=(os.environ['KIWI_USER'], os.environ['KIWI_PASS']))

    result = json.dumps({'status': 'ok', 'created': len(test_cases)})
    start_response('200 OK', [('Content-Type', 'application/json')])
    return [result.encode()]

⚙️GitHub Actions 集成

💡 各团队只需在现有 CI 末尾加入统一上报步骤，现有测试框架完全保留不变。

GitHub Secrets 配置（仓库级别）

Secret 名值
KIWI_URLhttps://kiwi.your-domain.com
KIWI_USERNAMEci-reporter（Kiwi 专用账号）
KIWI_PASSWORD（Kiwi 账号密码）

各团队 CI 模板（复制即用）

yaml — .github/workflows/test-and-report.yml
name: Test & Report
on: [push, pull_request]

env:
  KIWI_URL:      ${{ secrets.KIWI_URL }}
  KIWI_USERNAME: ${{ secrets.KIWI_USERNAME }}
  KIWI_PASSWORD: ${{ secrets.KIWI_PASSWORD }}
  KIWI_PLAN_ID:  "checkout-sprint-1"    # ← 各团队修改此处

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      # ──── 各团队保留自己的测试步骤 ────────────────────
      # Checkout 团队（Pytest）
      - run: pip install pytest && pytest --junit-xml=results.xml

      # Inventory 团队（Newman）替换上面一行为：
      # - run: npm install -g newman && newman run collection.json --reporters junit --reporter-junit-export results.xml

      # Customer 团队（Playwright）替换上面一行为：
      # - run: npx playwright test --reporter=junit
      # ──────────────────────────────────────────────────

      # 统一上报（所有团队相同，不需修改）
      - name: Upload results to Kiwi TCMS
        if: always()
        run: |
          pip install tcms-api
          python - <<'EOF'
          import os
          from tcms_api import TCMS
          tcms = TCMS(
            url=os.environ['KIWI_URL'],
            username=os.environ['KIWI_USERNAME'],
            password=os.environ['KIWI_PASSWORD']
          )
          tcms.upload_junit('results.xml', plan_id=os.environ['KIWI_PLAN_ID'])
          EOF

🤖AI 测试用例生成器

在阿里云可选择通义千问（成本更低）或继续用 Claude Haiku。

🇨🇳 通义千问 Qwen（推荐）

阿里云原生，无需跨境请求，价格更低（约 ¥0.04/千 token），账单统一

model: qwen-turbo

🌐 Claude Haiku

质量略高，需海外 API Key，每 Story 约 $0.01，函数计算内调用即可

model: claude-haiku-3-5

通义千问版本代码

python
from dashscope import Generation

def generate_test_cases(story_title, description, ac):
    response = Generation.call(
        model='qwen-turbo',
        api_key=os.environ['DASHSCOPE_API_KEY'],
        messages=[
            {
                "role": "system",
                "content": """你是电商平台高级QA工程师，
                根据用户故事生成JSON格式测试用例，
                包含正向/负向/边界/API场景。"""
            },
            {
                "role": "user",
                "content": f"Story: {story_title}\n描述: {description}\n验收: {ac}"
            }
        ]
    )
    return json.loads(response.output.text)
